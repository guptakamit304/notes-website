View and Blade in Laravel

Learn the basics of Views, Blade templating, and how to work with them in Laravel.

Views

Definition: Views in Laravel are files that contain HTML mixed with Blade templating syntax, which are used to display data to the user.

Location: Views are stored in the resources/views directory.

Returning Views

Definition: You return a view to the browser when responding to a request.

Usage:

  • return view('view_name');
  • You can pass data to views using:
  • return view('view_name', ['key' => 'value']);
  • Or with the with() method:
  • return view('view_name')->with('key', 'value');

Blade Templating

Definition: Blade is Laravel's templating engine that compiles templates into efficient PHP code. Blade files have a .blade.php extension.

Purpose: Blade simplifies syntax and enhances readability over traditional PHP templates.

Echoing Data

Definition: Blade provides an easy way to display dynamic data.

Usage:

  • {{ $variable }} to safely output data (automatically escapes HTML to prevent XSS).
  • {!! $variable !!} to output raw HTML (use cautiously).

Blade Directives

Definition: Blade directives are special tokens that provide control flow (like conditionals and loops) in templates.

Common Directives:

  • @if, @elseif, @else, @endif — Conditional statements.
  • @foreach, @for, @while, @forelse — Loop through arrays or objects.
  • @include('view_name') — Include another view.
  • @section('name') to define a section and @yield('name') to display the section in a layout.
