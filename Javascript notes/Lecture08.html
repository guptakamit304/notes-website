JavaScript Functions

Understanding Function Declarations and Hoisting

Function Declaration

  • A function declaration defines a named function that can be called throughout the code.
  • function functionName(parameters) {
    // code to execute
  }
  • Function declarations are hoisted to the top of their scope, so they can be called even before they appear in the code.
  • Functions can take parameters as placeholders for values, and arguments are the values passed to the function when it's called.

Function Hoisting

  • Hoisting moves declarations to the top of their scope.
  • Hoisting applies to both variable declarations and function declarations.

Important Notes

  • Function Expressions: Besides function declarations, JavaScript also allows function expressions. Example: const myFunction = function() {}
  • Scope and Closure: Functions create new scopes. Understanding closures is important for more complex applications.
  • Return Values: Functions can return values using the return statement. If no return statement is there it will return undefined by default.
