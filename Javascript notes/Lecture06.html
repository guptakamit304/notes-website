Error Handling

Managing Errors in JavaScript

Error Handling

JavaScript uses try...catch blocks to handle exceptions, throw for custom errors, and finally for cleanup. The Error object provides debugging information.

  • Try Catch Statement

    Wraps code that might throw an error in a try block. The catch block handles the error.

    try {
  // Code that may throw error
} catch (error) {
  // Handle error
} finally {
  // Optional cleanup
}
  • Finally Statement

    Optional block that runs after try/catch, for cleanup.

    try {
  // Code
} catch (error) {
  // Handle error
} finally {
  // Always runs
}
  • Throw Statement

    Used to create and raise custom errors.

    throw new Error("Something went wrong");

Types of Errors

  • ReferenceError: When accessing a non-existent variable.
  • TypeError: When a variable/parameter is not of the correct type.
  • RangeError: When a value is out of range.
  • SyntaxError: When there's a syntax mistake.
  • EvalError: Related to the eval() function (rarely used).

Important Notes

  • Error Object: The error object in catch provides details like error name and message for debugging.
  • Custom Error: You can create custom error objects with your own messages by using throw new Error('your message').
  • Error Propagation: If an error is not handled within a function, it will propagate to the calling function which might eventually crash your program if not handled properly.
