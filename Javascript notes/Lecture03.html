Variables and Scope in JavaScript

Variable Declaration

A variable is a named storage location that holds data values. It allows us to store and manipulate data efficiently during program execution.

JavaScript variables can store various types, and their values can change. Declaring a variable sets aside memory for the value.

Key Characteristics:
  • Name (Identifier): The unique name you use to refer to the variable in code.
  • Data Storage: Holds various types of values, from basic numbers to complex objects.
  • Mutable: Depending on how they're declared, variables can change values.

Declaring Variables

  • var: Function-scoped, can be re-declared and updated. Generally avoided in modern JavaScript. 
    var x = 10;
var x = 20; // Allowed
  • let: Block-scoped, cannot be redeclared but can be updated. 
    let y = 10;
y = 20; // Allowed
  • const: Block-scoped, cannot be redeclared or updated, unless it's an object. 
    const z = 10;
// z = 20; // Error
console.log(z); // 10

Variable Scope

Global Scope:

Variables declared outside a function or block can be accessed everywhere in the code.

var globalVar = "I am global";
function checkGlobal() {
  console.log(globalVar);
}
checkGlobal();
Local Scope:

Variables declared inside a function are only accessible inside that function.

function localScope() {
  var localVar = "I am local";
  console.log(localVar);
}
localScope();
// console.log(localVar); // Error: not defined
Block Scope:

Variables declared with let or const inside a block are accessible only within that block.

if (true) {
  let blockVar = "I am block scoped";
  console.log(blockVar);
}
// console.log(blockVar); // Error: not defined

Important Notes

  • Hoisting: var declarations are "hoisted" to the top of their scope, but assignments are not, leading to potential errors if used before declaration. let and const are also hoisted but are not initialized.
  • Best Practices: To write clear, modern code, prefer let for variables that need to change and const for those that should remain constant.
  • Naming Conventions: Always use clear, descriptive variable names, typically using camelCase (e.g., `myVariableName`).
