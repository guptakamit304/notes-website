Asynchronous JavaScript

Understanding Asynchronous Operations, Promises, and Async/Await

What is Asynchronous JavaScript?

Asynchronous JavaScript allows code to continue executing while waiting for long-running operations (like network requests) to complete. This prevents blocking the main thread, ensuring a smooth user experience.

Key Concepts

  • Callbacks: Traditional approach to handling asynchronous operations by passing a function to be called when the operation completes.
  • Promises: Objects that represent the eventual completion (or failure) of an asynchronous operation and allows for handling results.
  • Async/Await: Syntactic sugar built on top of Promises, making asynchronous code look and behave more like synchronous code.

Using Callbacks

Traditional method of handling async operations

function fetchData(callback) {
    setTimeout(() => {
       const data = "Data fetched successfully";
        callback(data);
   }, 1000);
}
fetchData(function(result) {
    console.log(result);
});

Working with Promises

Modern and cleaner approach for async operations

function fetchData() {
    return new Promise(resolve => {
        setTimeout(() => {
            const data = "Data fetched successfully";
            resolve(data);
        }, 1000);
    });
}

fetchData().then(result => {
    console.log(result);
});

Async/Await

Cleaner syntax using async and await to handle promises.

async function fetchData() {
    return new Promise(resolve => {
        setTimeout(() => {
            const data = "Data fetched successfully";
            resolve(data);
        }, 1000);
    });
}

async function getData(){
    const result = await fetchData();
    console.log(result);
}
getData()

Important Notes

  • Error Handling: Use try...catch blocks to handle errors in asynchronous code, particularly when using async/await.
  • Promise Chaining: When dealing with multiple async operations, you can create cleaner code by chaining multiple .then methods.
  • Concurrency vs Parallelism: Javascript does not support true parallelism. However the event loop in Javascript supports concurrency by executing other code while waiting for long running operations to complete.
