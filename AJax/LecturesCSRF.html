What is CSRF?

Definition:

CSRF allows attackers to perform unauthorized actions on behalf of an authenticated user by making a malicious request from another site the user is visiting. Since the user's browser automatically includes their credentials (like cookies) with the request, the web application mistakenly believes the request is legitimate.

Example:

If a user is logged into a banking website, an attacker could trick the user into visiting a malicious page that submits a request to transfer money from the user’s account to the attacker’s account.

How Does CSRF Work?

  • Malicious Request: An attacker creates a page or email with a link or script that makes an unauthorized request to a target website where the user is logged in.
  • Automatic Authentication: The browser automatically includes the user's session cookies (or other authentication tokens) with the malicious request, making the request appear legitimate to the website.
  • Unintended Action: Since the website does not distinguish between a legitimate user request and a malicious one, it performs the action (e.g., transferring money, changing account settings) without the user's consent.

Impact of CSRF

  • Unauthorized Actions: Attackers can make users unknowingly perform actions such as transferring funds, changing their email address or password, posting content, etc.
  • Data Corruption or Loss: Sensitive user data can be altered or lost due to an attacker's actions.
  • Privilege Escalation: In some cases, attackers can use CSRF to escalate their privileges or impersonate users, leading to further attacks.

Key Concepts in CSRF

  • Victim User: The user who is authenticated and unknowingly triggers the malicious request.
  • Attacker’s Website: The website or page controlled by the attacker, which contains the malicious code (e.g., form, link, or script).
  • Target Website: The website the user is logged into, which executes the malicious request because it assumes the request is legitimate.

Real-World Example of CSRF

Imagine you're logged into a bank website, and an attacker sends you an email with a link like this:

http://your-bank.com/transfer?amount=1000&to=attacker_account

If you click the link while still logged into your bank account, the bank processes the transfer request because it assumes it was made by you, the authenticated user.
