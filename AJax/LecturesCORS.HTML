What is CORS?

CORS (Cross-Origin Resource Sharing) is a security feature implemented by web browsers to prevent unauthorized cross-origin HTTP requests. It allows servers to specify which origins (domains) are permitted to access resources on the server.
By default, browsers enforce a same-origin policy, which restricts web pages from making requests to a domain different from the one that served the web page. CORS relaxes this restriction under controlled conditions.

Why Do We Use CORS?

  • Enable Cross-Origin Requests: Applications often need to fetch resources or communicate with APIs hosted on a different domain.
  • Enhance Security: CORS ensures that only authorized domains can access a server's resources.
  • Support Modern Web Development: With the rise of microservices, API-first designs, and cloud-hosted resources, it's common for a frontend and backend to reside on different domains.

Key Benefits of Using CORS

  • Controlled Resource Sharing: Only specified origins can access the server's resources.
  • Flexibility: APIs can serve multiple clients (e.g., web apps, mobile apps) hosted on different domains.
  • Compatibility with Modern Web Practices: Supports scenarios like single-page applications (SPAs), cloud services, and third-party integrations.

Common Use Cases for CORS

  • Frontend-Backend Communication: A React app on http://localhost:3000 needs to fetch data from an API at https://api.example.com.
  • Third-Party APIs: Accessing public APIs (e.g., Google Maps, OpenWeather API) from your web app.
  • CDN Resources: Loading scripts, styles, or assets from a Content Delivery Network (CDN) domain.
